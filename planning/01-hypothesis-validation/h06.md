# H6 — Under-lending forgoes measurable risk-adjusted revenue
**Group:** Value mechanisms (H5–H8) · **Tests assumption:** A4 (and informs A5, A10) · **Sub-problem:** P3 (and overall value case)

## Hypothesis statement (source §5)
The bank forgoes measurable risk-adjusted revenue from under-lending, net of expected loss and capital cost.

## Falsification condition (verbatim)
> "Net risk-adjusted uplift is immaterial after loss/capital costs"

## Validation method (source, verbatim)
Finance + Risk joint sizing using existing RAROC methodology.

**Method guard:** per A4 this is a business-level, RAROC-style analysis using the bank's **existing** methodology — not a new model. Gross figures are never presented without loss/capital/funding adjustments (R-BUS-01 mitigation: "Mandate risk-adjusted (not gross) sizing from day one").

## Required data
| Item | Use |
|---|---|
| M9 Portfolio financials per segment (revenue, loss, provisioning) | Baseline economics per segment |
| U8 Capital and funding cost allocation per segment | Precision of the risk-adjusted calculation (source purpose, verbatim) |
| M3 Offer data actually extended per customer | Gap between extended offers and policy-permitted ranges |
| M7 Risk appetite statement, policy ranges | The ceiling: headroom within which any uplift must fit (A5) |

## Analysis protocol (business/analytical terms)
1. **Joint ownership:** method agreed in a Finance + Risk working session (workplan wk5) using the bank's own RAROC (or equivalent) formula — the engagement does not introduce a new value formula.
2. **Comparison logic:** for populations evidenced as under-served (from H1–H3 outputs and M3-vs-M7 headroom), estimate incremental risk-adjusted revenue of offers moved *within existing policy ranges*: gross revenue uplift − expected loss delta − capital cost delta − funding cost delta.
3. **Constraints applied:** risk appetite headroom (A5, OQ-06) as a hard cap; affordability as a hard constraint (§6.1 overshoot mitigation; §6.3 responsible lending) — populations failing affordability are excluded from the sized opportunity, not discounted.
4. **Scenario logic (A10):** per-segment win-win test — value must be shown net-positive for both the customer-outcome and the portfolio-risk view per segment, not asserted in aggregate; include a downside scenario (E3 requires "net value range documented, including downside scenario").
5. **Output form:** value ranges (not point estimates) by sub-problem (P1/P2/P3) and by opportunity (O1–O8 mapping in D5); explicitly reproducible by Finance.
